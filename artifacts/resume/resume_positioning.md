# Resume Positioning

## Project title

**PUNCHLINE — Real-Time Multiplayer Party Game Platform**

## One-line project summary

Built a server-authoritative multiplayer party game platform with WebSocket room orchestration, reconnectable guest sessions, phase-based game state machines, PostgreSQL-backed card/deck storage, async daily rounds, and an AI-assisted content pipeline for generating and moderating playable decks.

## Strong 3-bullet version

- Built a Go + PostgreSQL real-time multiplayer game backend with WebSocket room orchestration, reconnectable guest sessions, server-authoritative phase transitions, rotating judges, submissions, scoring, and durable game results.
- Designed a room actor/state-machine architecture for live games, modeling lobby, dealing, submitting, judging, scoring, and finished phases with validated player commands and broadcast room snapshots.
- Extended the platform with async daily rounds and a content system for card packs, AI-assisted generation jobs, safety tiers, card ratings, reports, and moderation-ready publishing workflows.

## Shorter ATS version

- Built a real-time multiplayer party game platform using Go, PostgreSQL, Next.js, TypeScript, and WebSockets, with server-authoritative room state, reconnectable guest sessions, scoring, async daily rounds, and AI-assisted card/deck generation workflows.

> A multiplayer systems project disguised as a party game.

The product surface is simple, but the engineering surface is meaningful.
